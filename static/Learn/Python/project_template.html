Introduction

Welcome to our exciting project that demonstrates how to automate the creation of a GitHub repository using Python and the GitHub API.

In today's digital age, managing and organizing code repositories is an essential skill for developers, and leveraging the power of APIs can streamline this process significantly. This project will guide you through the process of programmatically creating a repository on GitHub with a few lines of Python code.

Imagine you're working on a series of projects or need to quickly set up repositories for multiple components of a larger application. Instead of manually going through the GitHub website to create each repository, you can use this script to automate the task, saving you valuable time and effort.

The script starts by importing the requests library, which is a powerful tool for making HTTP requests in Python. You will learn how to generate and use a personal access token from GitHub, which is crucial for authenticating your API requests securely. This token acts as a password that allows your script to interact with your GitHub account without exposing your actual password.

The script prompts you to enter a repository name, making it flexible for creating repositories dynamically based on your input. The core functionality is encapsulated in the create_repository function. This function constructs the appropriate API endpoint URL and sets up the necessary headers and payload for the request. By using the POST method, it sends a request to GitHub to create a new repository with the specified name and initializes it with a README file.

Upon execution, the script handles the response from GitHub, providing feedback on whether the repository was created successfully or if there were any issues. This project not only teaches you how to interact with web APIs but also emphasizes best practices for handling API responses and error checking.

By the end of this project, you will have a solid understanding of how to automate repository creation on GitHub, which can be a valuable addition to your development toolkit. Whether you're managing personal projects, collaborating with a team, or contributing to open-source, this script offers a streamlined and efficient way to manage your repositories.

Dive into this project and discover the convenience of automating routine tasks with Python and the GitHub API. Happy coding!

Automating GitHub Repository Creation

Welcome to our project! This project demonstrates how to automate the creation of GitHub repositories using Python and the GitHub API.

            
import requests

# Your GitHub personal access token, generate from - https://github.com/settings/tokens
access_token = 'ghp_fmVAzn5snyK6EcCygRh9xQwJinATxxxxxxxx'

# The owner of the repository (your GitHub username or organization)
owner = 'name here'

# The name of the repository
repo_name = input("Enter repo name : ")

# Create a new repository
def create_repository():
    url = f'https://api.github.com/user/repos'
    headers = {'Authorization': f'token {access_token}'}
    payload = {'name': repo_name, 'auto_init': True}
    response = requests.post(url, headers=headers, json=payload)
    if response.status_code == 201:
        print(f'Repository "{repo_name}" created successfully.')
    else:
        print(f'Failed to create repository. Status code: {response.status_code}')
        print(f'Response content: {response.text}')

# Call the function to create or add the file
create_repository()

Explanation: This script automates the creation of a GitHub repository using Python and the GitHub API. It utilizes the `requests` library to send HTTP requests to the GitHub API, including authentication with a personal access token. The script prompts the user to enter a repository name and then calls the `create_repository` function to create the repository on GitHub.
